# 阿里云部署指南 - 中山市中小学生健康宣传互动平台

## 方案一：使用自动部署脚本（推荐）

### 前提条件
1. 阿里云 ECS 服务器（建议配置：1核2G 以上，CentOS 7+ 或 Ubuntu 20.04+）
2. 已开放安全组端口：22(SSH)、80(HTTP)、443(HTTPS)、3000(应用)
3. 本地已配置 SSH 密钥登录

### 部署步骤

```bash
# 1. 进入项目目录
cd /workspace

# 2. 运行部署脚本
./deploy.sh <你的服务器IP>

# 示例
./deploy.sh 123.45.67.89
```

脚本会自动完成：
- 安装 Docker 和 Docker Compose
- 上传项目文件
- 构建并启动容器
- 可选配置 Nginx 和 SSL

## 方案二：手动部署

### 步骤 1：购买阿里云服务器
1. 登录 [阿里云控制台](https://ecs.console.aliyun.com/)
2. 创建 ECS 实例：
   - **地域**：选择靠近用户的地域（如华南1-深圳）
   - **实例规格**：1核2G 或更高
   - **镜像**：Ubuntu 22.04 LTS
   - **带宽**：3-5Mbps
   - **安全组**：开放 22、80、443、3000 端口

### 步骤 2：连接服务器
```bash
ssh root@<你的服务器IP>
```

### 步骤 3：安装 Docker
```bash
# 更新系统
apt-get update && apt-get upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 启动 Docker
systemctl start docker
systemctl enable docker

# 安装 Docker Compose
curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
```

### 步骤 4：上传项目代码
```bash
# 在本地执行
scp -r /workspace/backend /workspace/frontend /workspace/Dockerfile /workspace/docker-compose.yml root@<服务器IP>:/opt/zhongshan-health-app/
```

### 步骤 5：启动服务
```bash
# 在服务器上执行
cd /opt/zhongshan-health-app
mkdir -p data uploads/works

# 构建并启动
docker-compose up -d --build

# 查看日志
docker-compose logs -f
```

### 步骤 6：配置域名（可选）
```bash
# 安装 Nginx
apt-get install -y nginx

# 创建配置文件
cat > /etc/nginx/sites-available/health-app << 'EOF'
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }

    location /uploads {
        alias /opt/zhongshan-health-app/uploads;
        expires 30d;
    }

    client_max_body_size 50M;
}
EOF

# 启用配置
ln -s /etc/nginx/sites-available/health-app /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t && systemctl restart nginx
```

### 步骤 7：配置 HTTPS（强烈推荐）
```bash
# 安装 Certbot
apt-get install -y certbot python3-certbot-nginx

# 申请证书
certbot --nginx -d your-domain.com

# 自动续期已配置，无需额外操作
```

## 方案三：阿里云容器服务（K8s）

适合高并发场景，需要阿里云 ACK 集群。

```yaml
# k8s-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: health-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: health-app
  template:
    metadata:
      labels:
        app: health-app
    spec:
      containers:
      - name: health-app
        image: your-registry.cn-shenzhen.aliyuncs.com/zhongshan/health-app:latest
        ports:
        - containerPort: 3000
        volumeMounts:
        - name: data
          mountPath: /app/backend/data
        - name: uploads
          mountPath: /app/backend/uploads
      volumes:
      - name: data
        persistentVolumeClaim:
          claimName: health-app-data
      - name: uploads
        persistentVolumeClaim:
          claimName: health-app-uploads
---
apiVersion: v1
kind: Service
metadata:
  name: health-app-service
spec:
  selector:
    app: health-app
  ports:
  - port: 80
    targetPort: 3000
  type: LoadBalancer
```

## 常见问题

### Q: 端口访问不了？
A: 检查阿里云安全组规则，确保端口已开放。

### Q: 如何绑定微信公众号？
A: 在公众号后台「功能设置」-「网页授权域名」中配置你的域名。

### Q: 数据库文件在哪里？
A: 在服务器 `/opt/zhongshan-health-app/data/health_app.db`

### Q: 上传的文件在哪里？
A: 在服务器 `/opt/zhongshan-health-app/uploads/works/`
